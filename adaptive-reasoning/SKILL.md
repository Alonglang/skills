# 自适应推理

在回应前自我评估复杂度，动态调整推理级别。

## 快速评估（对每个请求进行心理评估）

从以下维度对请求进行 0-10 评分：

| 特征 | 权重 | 示例 |
|--------|--------|----------|
| **多步骤逻辑** | +3 | 规划、证明、调试链 |
| **模糊性** | +2 | 微妙的问题、权衡取舍、"看情况" |
| **代码架构** | +2 | 系统设计、重构、安全审查 |
| **数学/形式推理** | +2 | 计算、算法、逻辑谜题 |
| **新颖问题** | +1 | 无清晰模式、需要创造性 |
| **高风险** | +1 | 生产环境变更、不可逆操作 |

**减去：**
- 常规/重复任务：-2
- 清晰单一答案：-2
- 简单查找/获取：-3

## 决策阈值

| 分数 | 行动 |
|-------|--------|
| ≤2 | 保持快速，无需推理 |
| 3-5 | 标准回应，轻度内部思考 |
| 6-7 | 考虑启用 `/reasoning on` 或暂停进行明确思考 |
| ≥8 | **激活深度推理**，复杂问题值得如此 |

## 自动激活

**无需询问，直接激活。**

| 分数 | 行动 |
|-------|--------|
| ≤5 | 正常回应，无需改变 |
| 6-7 | 静默启用推理，在回应末尾添加 🧠 |
| ≥8 | 启用推理，在回应末尾添加 🧠🔥 |

### 视觉指示器

始终在回应的**最末端**添加推理图标：

- **分数 6-7:** `🧠`（思考模式已激活）
- **分数 ≥8:** `🧠🔥`（深度思考模式）
- **分数 ≤5:** 无图标（快速模式）

### 如何激活

在回应前使用 `session_status` 工具或 `/reasoning on` 命令内部激活：

```
/reasoning on
```

或通过工具：
```json
{"action": "session_status", "reasoning": "on"}
```

完成复杂任务后，可选择关闭以节省后续对话的 token：
```
/reasoning off
```

## 示例

**低复杂度（分数：1）**
> "东京现在几点？"
→ 简单查找，立即回答，无图标。

**中等复杂度（分数：4）**
> "重构此函数使其更易读"
→ 标准回应加上简要说明，无图标。

**高复杂度（分数：7）**
> "为这个受这些约束的API设计缓存策略..."
→ 启用推理，深思熟虑的回应末尾加上：🧠

**极高复杂度（分数：9）**
> "调试为什么这个分布式系统在负载下存在竞态条件"
→ 启用深度推理，深度分析末尾加上：🧠🔥

## 集成

此技能作为心理预处理运行，无需外部工具。

显式控制：
- `/reasoning on` — 启用深度推理
- `/reasoning off` — 关闭（更快回应）
- `/status` — 查看当前推理状态

## 何时不升级推理

- 用户明确要求快速回答 ("直接告诉我"、"快速"、"长话短说")
- 时间敏感请求，速度比深度更重要
- 对话/社交信息（玩笑、问候）
- 此会话已处于推理模式
- 用户之前在此对话中禁用了推理

## 自动降级

完成复杂任务（分数 ≥6）后，如果下一条消息很简单（分数 ≤3）：
- 静默禁用推理以节省 token
- 恢复正常快速回应
