# secucheck - OpenClaw安全审计

OpenClaw部署的综合审计技能。分析配置、权限、暴露风险和运行环境，提供上下文感知的建议。

---

## 摘要

**secucheck** 对OpenClaw设置执行只读安全审计：

- **7个审计领域**：运行时、通道、代理、计划任务、技能、会话、网络
- **3个专业级别**：初学者（类比）、中级（技术）、专家（攻击向量）
- **上下文感知**：考虑VPN、单用户、自托管场景
- **运行时检查**：实时系统状态（网络暴露、容器、权限）
- **仪表板**：带安全分数的可视化HTML报告
- **本地化输出**：最终报告匹配用户语言

**从不自动修改配置。** 所有修复都需要明确的用户确认。

## 快速开始

### 安装
```bash
clawhub install secucheck
```

### 使用
询问你的OpenClaw代理：
- "安全审计"
- "secucheck"
- "运行安全检查"

### 专业级别
当提示时，选择你的级别：
1. **初学者** - 简单类比，无术语
2. **中级** - 技术细节，配置示例
3. **专家** - 攻击向量，边缘情况，CVE

所有级别运行相同的检查 - 只是解释深度不同。

## 执行流程

### 步骤1：询问级别（运行前）

用用户的语言展示选项。示例（中文）：

```
你希望有多详细的技术说明？

1. 🌱 初学者 - 我会用类比简单地解释
2. 💻 中级 - 技术细节和配置示例
3. 🔐 专家 - 包括攻击向量和边缘情况

📌 所有级别运行相同的检查 - 只是解释深度不同。
```

**在此停止。等待用户响应。**

### 步骤2：运行审计

```bash
bash ~/.openclaw/skills/secucheck/scripts/full_audit.sh
```

返回按严重性分类的结果JSON。

### 步骤3：格式化输出

解析JSON输出并根据用户的专业级别格式化。**最终报告必须使用用户语言。**

#### 报告结构（按类别组织）

```
🔒 安全审计结果

📊 摘要表格
| 严重性 | 数量 |
|---------|------|
| 🔴 严重 | X |
| ...

⚡ 运行时
- [与运行时类别相关的发现]

🤖 代理  
- [与代理类别相关的发现]

📁 工作区
- [与工作区类别相关的发现]

🧩 技能
- [与技能类别相关的发现]

📢 通道
- [与通道类别相关的发现]

🌐 网络
- [与网络类别相关的发现]
```

按 `category` 字段组织发现，而不仅是严重性。在每个类别内，显示严重性图标并解释。

### 步骤4：自动打开仪表板

文本报告后，自动生成并提供仪表板：

```bash
bash ~/.openclaw/skills/secucheck/scripts/serve_dashboard.sh
```

脚本返回带有 `url`（局域网IP）和 `local_url`（localhost）的JSON。**使用 `url` 字段**（不是localhost），因为用户可能从其他设备访问。

示例：
```
📊 仪表板已打开：http://192.168.1.200:8766/secucheck-report.html
```

## 风险级别

```
🔴 严重 - 需要立即采取行动。可能发生主动利用。
🟠 高     - 重大风险。应尽快修复。
🟡 中等   - 值得关注。计划解决。
🟢 低      - 小问题或最佳实践建议。
⪪ 信息     - 不是风险，但值得记录。
```

## 风险矩阵

```
                工具权限
              最小       完整
          ┌──────────┬──────────┐
 暴露   低  │   🟢     │   🟡     │
            │  安全    │  谨慎   │
          ├──────────┼──────────┤
          │   🟡     │   🔴     │
        高  │  谨慎    │  严重   │
          └──────────┴──────────┘

暴露 = 谁可以与机器人对话（DM策略、群组访问、公开通道）
工具权限 = 机器人可以做什么（执行、文件访问、消息传递、浏览器）
```

## 应用修复

**关键规则：**

1. **从不自动应用修复。** 始终先显示建议。
2. **警告功能影响。** 如果修复可能破坏某些功能，请说明。
3. **获取明确的用户确认** 任何配置更改之前。

示例流程：
```
代理： "更改此设置将禁用#dev频道中的exec。
       如果你在那里使用代码执行，它将停止工作。
       应用此修复？"
用户： "是"
代理： [通过gateway config.patch应用修复]
```

## 语言规则

- **内部处理**：始终英语
- **思考/推理**：始终英语
- **最终用户报告**：匹配用户语言
- **技术术语**：保持英文（exec、cron、gateway等）
